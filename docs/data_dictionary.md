# Data Dictionary

The sample dataset is modeled and deterministic for public demo use. It is generated by `scripts/generate_sample_data.py`, saved to `data/sample_business_data.csv`, and loaded into the SQLite table `business_metrics`.

## Table: business_metrics

| Column | Type | Description |
| --- | --- | --- |
| id | Integer | SQLite primary key generated when data is loaded. |
| month | Date text | First day of the month for the business record. |
| region | Text | Sales region: Northeast, Midwest, South, or West. |
| product_category | Text | Product or service line: Analytics Software, Implementation Services, Training Workshops, or Support Plans. |
| customer_segment | Text | Customer group: Small Business, Mid-Market, or Enterprise. |
| marketing_spend | Real | Marketing dollars allocated to the row. |
| operating_cost | Real | Operating cost assigned to the row, excluding marketing spend. |
| units_sold | Integer | Units sold for the month, region, product category, and segment. |
| average_price | Real | Average selling price per unit. |
| revenue | Real | Units sold multiplied by average price. |
| gross_margin_rate | Real | Gross margin rate as a decimal. Example: 0.42 means 42%. |
| gross_margin | Real | Revenue multiplied by gross margin rate. |
| profit | Real | Gross margin minus operating cost and marketing spend. |
| retention_rate | Real | Estimated customer retention rate as a decimal. |
| churn_risk | Real | Estimated churn risk as a decimal, calculated as 1 minus retention rate. |
| created_at | Date text | Timestamp added by SQLite when each row is loaded. |

## Dataset Grain

Each row represents one monthly business outcome for a unique combination of:

- Month
- Region
- Product category
- Customer segment

The generated dataset contains:

- 36 months
- 4 regions
- 4 product categories
- 3 customer segments
- 1,728 total records

## Synthetic Data Assumptions

- Product categories have different base unit volume, prices, and margins.
- Regions have different demand, cost, and retention profiles.
- Customer segments affect demand, price, and margin.
- Seasonality and random-but-deterministic variation make trends more realistic.
- The data is safe for public screenshots and should not be treated as real financial data.
